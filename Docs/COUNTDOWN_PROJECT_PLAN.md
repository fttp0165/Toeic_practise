# 倒數天數 →「可開啟的衝刺專案」— 功能規劃書

> 把目前寫死的「20 天衝刺」改造成**可開啟、可設定的衝刺專案**：
> 使用者自己開一個專案、選考試日期（或直接輸入倒數幾天），系統據此產生整份倒數計畫與間隔複習排程。
>
> 本文件只談規劃與設計決策，尚未動任何程式碼。對應現況前端：`index.html` + `app-core.js`。

---

## 1. 動機與目標

現在的衝刺頁是**單一、寫死 20 天**的計畫：只能設「開始日」，天數固定 20、階段邊界固定、最後背新字日固定第 18 天。真實情況是：

- 每次報考，距離考試的天數不一樣（可能 12 天、30 天、45 天）。
- 一年可能考好幾次，想各自有一份獨立的倒數計畫與紀錄。
- 使用者的心智模型是「**我距離考試還有幾天**」，而不是「我是第幾天」。

**目標**：把「20 天衝刺」升級為**衝刺專案（Sprint Project）**——

1. 可**新建 / 開啟 / 切換 / 刪除**多個具名專案（比照現有「多單字庫」的模式）。
2. 每個專案可設定**考試日期**，系統自動換算**倒數天數**與 Day 1；也允許直接輸入「倒數 N 天」。
3. 五階段計畫、任務量、最後背新字日、間隔複習排程，全部**依專案天數自動縮放**，短計畫也合理。
4. 舊使用者的既有資料（單一 `toeic20_start` + 任務 + 單字）**無痛遷移**成「預設專案」，不遺失。

## 2. 名詞定義

| 名詞 | 意義 |
|---|---|
| **衝刺專案（Project）** | 一次「倒數到某場考試」的計畫：有名稱、開始日、考試日 / 總天數，及其專屬的每日任務完成狀態。 |
| **倒數天數（Countdown）** | `考試日 − 今天`，顯示「距離考試還有 N 天」。 |
| **總天數（totalDays）** | `考試日 − 開始日 + 1`，一份計畫共幾個 Day，取代目前寫死的 `TOTAL=20`。 |
| **最後背新字日（lastNewDay）** | 專案內最後一天排新單字；其後為模擬考收尾，取代寫死的 `LAST_NEW=18`。 |

## 3. 現況與「寫死 20 天」的所有位置

改造前要先盤點所有綁死 20 / 18 的點（皆在 `app-core.js`，除非另註）：

| 位置 | 現況 | 影響 |
|---|---|---|
| `const TOTAL=20, LAST_NEW=18`（第 2 行） | 全域常數 | 幾乎所有天數邏輯的根 |
| `phaseOf(d)`（第 7–13 行） | 邊界 `d<=3 / 9 / 14 / 18` 寫死 | 五階段只對 20 天正確 |
| `tasksOf(d)`（第 16–50 行） | 用相同天數邊界分派任務 | 任務量綁死 20 天 |
| `newvocab` 任務 `d<=LAST_NEW`（第 46 行） | 最後背新字日 = 18 | 短/長計畫錯位 |
| `todayDay()`（第 349–357 行） | `diff>TOTAL → 99`、單一 `startDate` | 完成判定綁死 20；無專案概念 |
| `renderStrip()`（第 369–386 行） | `for d=1..TOTAL` | 日期條固定 20 格 |
| `renderDay()`（第 458 行） | `<span class="of"> / 20</span>` | 標題寫死「/ 20」 |
| `waveHTML` / `recallHTML` / `recallBatches`（第 53–58、388–430 行） | 用 `LAST_NEW` 濾批次 | 間隔複習範圍綁死 |

| `index.html` 第 31–36 行 | 只有一個 `<input id="start">` | UI 無專案選擇 |
| 匯出/匯入（第 936、941 行）、`getLocal/applyRemote`（第 1172–1190 行） | 只帶 `start` 單值 | 需擴充成專案集合 |

## 4. 設計方案

### 4.1 資料模型（新增，比照多單字庫）

新增專案集合，沿用現有「陣列清單 + 目前選取鍵」的既有模式（如 `libs` / `curlib`）：

```js
// localStorage key: "toeic20_projects"（隨雲端同步）
projects = [
  {
    id: "p1700000000",   // 建立時的穩定亂數/時戳字串
    name: "3 月正式考",
    start: "2026-03-01",  // Day 1
    exam:  "2026-03-20",  // 考試日（可空；空則用 days）
    days:  20,            // 總天數；由 exam−start 推算，或手動輸入
    tasks: { "d3:gram": true, ... }  // 該專案的任務完成狀態（原 toeic20_tasks）
  },
  ...
]

// localStorage key: "toeic20_curproj"（目前開啟的專案 id）
```

**關鍵設計取捨**：把原本全域的 `toeic20_tasks` **移進各專案的 `tasks`**，讓每個專案的每日打勾互相獨立。開始日與天數同理內化到專案。

### 4.2 考試日 ↔ 倒數天數

- 主要輸入是**考試日期**：`days = 天(exam) − 天(start) + 1`；倒數 = `天(exam) − 今天`。
- 也提供「直接輸入倒數 N 天」：以今天為 Day 1，`exam = today + (N−1)`。
- 顯示層：把現有「Day X / 20」改為「**Day X / {days}**」，並在頂部加一行「**距離考試還有 N 天**」。

### 4.3 五階段自動縮放（取代寫死邊界）

把五階段改成**比例**而非絕對天數，依 `days` 算邊界（無條件進位、每段至少 1 天、收尾段至少 2 天）：

| 階段 | 佔比 | 20 天時 | 12 天時（示意） |
|---|---|---|---|
| 打地基 | 15% | Day 1–3 | Day 1–2 |
| 聽力＋文法主攻 | 30% | Day 4–9 | Day 3–6 |
| 擴張 Part 3/4/6 | 25% | Day 10–14 | Day 7–9 |
| 攻 Part 7＋時間 | 20% | Day 15–18 | Day 10 |
| 全真模擬＋收尾 | 10%（≥2 天） | Day 19–20 | Day 11–12 |

- `phaseOf(d, days)` 依上表算出各段邊界後回傳階段（**20 天時須與現行結果完全一致**，作為回歸測試基準）。
- `lastNewDay = days − 2`（收尾段不背新字；短到 `days≤3` 時退化為 `days−1`，至少留 1 天收尾）。
- `tasksOf(d, days)` 改成**先算階段、再給該階段任務**，不再直接比對絕對天數。

### 4.4 間隔複習（維持核心，不改行為）

- `INTERVALS=[1,3,7,14]` 不變；`recallBatches(d)` 改用 `lastNewDay` 取代 `LAST_NEW` 過濾。
- 短計畫（如 7 天）`+14` 自然不會命中——現有 filter 已能處理，無需特例。
- **靈魂不變**：先回想、再翻開驗證（輸出 > 輸入），任何改動都不得改成被動瀏覽。

### 4.5 UI 變更（`index.html` + 渲染）

1. 「開始日」那一列（第 31–36 行）換成**專案列**：
   - 專案下拉（開啟 / 切換）、「＋ 新增專案」、「重新命名 / 刪除」。
   - 選定專案後顯示：名稱、開始日 `date`、考試日 `date`（或倒數 N 天輸入）、「距離考試還有 N 天」。
2. 尚無任何專案時，顯示引導卡：「建立你的第一個衝刺專案，選好考試日就開始倒數」。
3. 日期條、當日卡、進度總覽全部改吃「目前專案」的天數與任務。

### 4.6 單字池的歸屬（已定案：持續累積、永不清空）

**決策（使用者已確認）**：單字池**跨所有專案共用、只增不清、持續累積**。單字是長期資產，不隨任何一個專案結束或刪除而消失。

具體規則：

- `vocab`（含衝刺批次與 `lib` 桶）維持**單一全域共用池**，衝刺批次仍以天數為 key。
- **刪除專案時，一律不動 `vocab`**；只移除該專案的設定與任務打勾狀態。
- **清除進度 / 重置**，也**不得清空單字池**——重置只影響專案設定與任務，單字與學習進度（`lo`/`ri`/`pr`/`da`）一律保留。
- 多專案並存時的已知限制：兩個同時進行的專案會**共享同一個「第 N 批」**（day-N key 相撞）。因為單字本來就要跨專案累積，這通常反而是想要的行為；若日後真的需要各專案批次完全隔離，再升級為批次 key 加專案前綴（如 `"p1700000000:3"`）——列為 §7 的 M3（選做），不影響「單字持續累積」的原則。

## 6. 資料遷移（不可遺失既有資料）

啟動時做一次性遷移（偵測「有舊 key、無 `toeic20_projects`」時）：

1. 若存在舊 `toeic20_start`，建立一個預設專案：
   `{ id, name:"我的衝刺", start: 舊 start, days: 20, tasks: 舊 toeic20_tasks }`。
2. 設 `toeic20_curproj` = 該預設專案 id。
3. 舊 `toeic20_start` / `toeic20_tasks` 保留讀取相容（僅遷移、不即刻刪除，降低風險）。
4. `vocab` / `libs` / `pr` 等**一律不動**（單字池持續累積、永不清空——見 §4.6）。
5. 匯入舊版 JSON（只有 `start`/`tasks`）時，`load()` 自動包成單一預設專案。

原則：遷移只新增、不破壞；一律可回退到舊資料。

## 7. 分階段開發藍圖

| 里程碑 | 內容 | 產出 |
|---|---|---|
| **M1｜可設定的單一專案** | 抽掉寫死常數，改吃「目前專案」的 `start/days`；階段與任務自動縮放；UI 加考試日 + 倒數顯示；一次性遷移。**先不做多專案切換**，內部就緒即可。 | 使用者能自訂天數/考試日並正確倒數 |
| **M2｜多專案開啟/切換** | 專案 CRUD、下拉切換、命名、刪除；匯出/匯入與雲端同步帶上 `projects`。 | 可開好幾份計畫各自獨立 |
| **M3｜批次隔離（選做，非必要）** | 單字池「持續累積、不清空」已定案；此里程碑僅在日後真的需要「各專案批次互不干擾」時，才把衝刺批次 key 加專案前綴。**不影響單字累積原則。** | 多專案批次不互相干擾 |
| **回歸驗證** | 以「20 天、`LAST_NEW=18`」為基準：`phaseOf`/`tasksOf`/`recallBatches` 輸出須與現行**逐日一致**。 | 保證不回退現有行為 |

建議每個里程碑一個 commit（沿用專案「一功能一 commit」慣例）。

## 8. 風險與相容

- **單字絕不遺失（最高優先）**：新增/刪除專案、清除進度、重置，任何路徑都**不得清空 `vocab` 與學習進度**。實作時把「刪專案」「重置」與「單字池」完全解耦，並加防呆。
- **相容第一**：schema 只增不改語意；20 天既有計畫遷移後行為須**逐日等價**。
- **短計畫退化**：`days` 很小（≤4）時，五階段會壓縮甚至合併——需定義下限（每段≥1、收尾≥2、`days≥3` 才有意義），並在極短時給合理提示。
- **雲端同步**：`sync.js` 只依賴 `window.TOEIC` 介面，負載加欄位即可，不動同步層。
- **多專案撞 key（方案 A 已知限制）**：文件內明示「批次單字目前跨專案共用」，避免使用者誤解。
- **無建置**：維持純靜態、可直接開瀏覽器，不引入框架/打包。
