# TOEIC 20 天衝刺追蹤器 — 專案規劃書

> 個人用的 20 天 TOEIC 衝刺工具，核心是「海馬迴間隔複習」單字法。
> 目標分數：300 → 500。目前為單檔靜態網頁，計畫逐步加入備份與帳號同步。

---

## 1. 專案概述

| 項目 | 內容 |
|---|---|
| 產品定位 | 個人 20 天 TOEIC 衝刺的每日任務 + 單字間隔複習工具 |
| 使用者 | 單一使用者（自己），非多人註冊平台 |
| 學習目標 | TOEIC 300 → 500，重點在「撿回該拿沒拿的分」而非提升英文天花板 |
| 目前型態 | 三頁靜態網站（`index.html` 衝刺頁、`library.html` 單字庫頁、`dashboard.html` 練習紀錄頁），共用 `styles.css` / `app-core.js` / `sync.js`，原生 JS、無框架、無建置流程 |
| 儲存方式 | 瀏覽器 `localStorage`（綁定裝置 + 瀏覽器）＋ 登入後 Firebase RTDB 雲端同步 |
| 代管 | GitHub Pages（repo: `fttp0165/Toeic_practise`，public） |
| 線上網址 | `https://fttp0165.github.io/Toeic_practise/`（啟用 Pages 後） |

**設計原則**：先用最輕的方式把工具跑起來，不要為了一個讀書工具過度工程化。帳號/同步是「需求出現後再加」，不是預設。

## 2. 現況：已完成功能（v1，單檔 HTML）

以下功能已實作於現有 `index.html`：

- **開始日設定**：使用者設定 Day 1 日期，系統自動換算今天是第幾天（1–20，未開始 / 已完成有對應狀態）。
- **日期選擇條**：橫向 1–20 天，顯示每天完成度 `x/total`，今天有外框標示，可點選查看任一天。
- **當日卡片**：大字日數、階段徽章與說明、「記憶波次」視覺（+1 / +3 / +7 / +14 哪幾段今天到期會點亮）。
- **間隔複習區（核心）**：自動列出今天到期的舊批次，把該批已輸入的單字攤成**翻卡**——顯示英文 + 例句、中文意思預設模糊，點按才翻開（主動回想）。
- **每日任務清單**：依階段帶具體練習量（題數 / 句數 / 篇數），可打勾、狀態持久化。
- **睡眠固化提醒** 與 **「只有 1 小時」縮減版** 提示。
- **每日單字輸入**：欄位為單字 / 中文意思 / 例句 / 備註（可留空），計數 `x/15`，可刪除；輸入後自動排進往後的間隔複習。整庫去重（跨天、不分大小寫），重複輸入只把新例句補進既有的字、不另建、不重排批次。
- **編輯單字**：每個單字可就地編輯（單字 / 中文 / 多例句各含英文＋中文翻譯 / 備註）；改名撞到既有字會被擋下，避免破壞去重。
- **語音朗讀（TTS）**：單字與例句旁有 🔊 鈕（免後端、免金鑰）。**單字**優先抓 Free Dictionary API 的真人發音 mp3、抓不到才退回 Web Speech；**例句**用 Web Speech（自動挑較好的英文聲音，如 Google US／Natural／Online）。翻卡上的朗讀鈕不會誤觸翻面。
- **單字庫總覽**：可收合卡片列出全部單字（跨天、附「第 N 批」標籤），支援搜尋（單字／中文／例句／備註）與「列表 / 翻卡」兩種檢視。
- **單字庫翻卡複習**：總覽切到「翻卡」即把整個單字庫攤成翻卡，中文預設模糊、點按翻開，可隨時自由複習（與晨間到期複習互補）。
- **海馬迴單字庫頁面（獨立頁 `library.html`）**：頂部導覽列在「20 天衝刺(`index.html`) / 海馬迴單字庫(`library.html`)」兩頁間切換。單字庫頁與 20 天**共用同一池單字**（同一份 `localStorage`／雲端資料），但複習排程獨立、依**真實日期**跑 +1／+3／+7／+14（每字記 `lo`/`ri`）。可把任一字選為「今天新學」、新增單字進庫（存入 `lib` 桶並開始學習），頁面顯示今天到期複習翻卡與「記得／忘了」進階。完全不影響 20 天批次與進度。「全部單字」另有「列表／翻卡」切換可整庫自由翻卡。
- **單字測驗模式**：單字庫頁可選**加入日期**(`da`)做測驗，考那天加入的字(目前庫，預設今天)；用不可變的加入日期分組，按「今天新學」不會把字搬到今天。**混合出題**——看中文輸入英文（trim＋不分大小寫比對）或看英文四選一選中文；即時對錯回饋、顯示正解，結束顯示得分。純測驗、不改 SRS 排程。
- **多單字庫（分主題）**：單字庫頁可建立／切換／刪除多個具名單字庫（以 `word.lib` 標籤分庫），頁面所有區塊（統計／今天複習／測驗／逐字紀錄／單字清單）都隨選取的庫切換。20 天衝刺的字維持在衝刺頁、不混入。
- **匯入／匯出**：每個單字庫可匯出 **CSV**（單字／中文／例句／備註，Excel 友善）；可一鍵匯出 **JSON 完整備份**（含開始日／任務／單字／學習進度／紀錄／庫清單）。匯入同支援 CSV（加進目前庫、去重）與 JSON（還原整份備份）。
- **練習紀錄頁（`dashboard.html`，獨立頁）**：所有單字庫合計，顯示今天與累計的「練習數／答對／答錯」與**正確率**，近 14 天**對錯堆疊長條圖**（綠＝對、琥珀＝錯）；下方「逐字練習紀錄」依日期分組，列出每個字當天的練習／對／錯（例：`bookstore 6/16 練習5 對3 錯2`）。資料逐字記在 `pr`、隨雲端同步。三頁頂部導覽列互連。
- **進度總覽**：總完成率 %、完成項數、連續完成天數、20 格完成度方格圖。
- **方法與計畫說明**：海馬迴四原則、20 天五階段地圖（可收合）。
- **清除進度**：一鍵重置 localStorage。

## 3. 核心邏輯規格（間隔複習引擎）

這是整個工具的靈魂，遷移 / 重構時必須維持行為一致。

```
常數
  TOTAL_DAYS = 20
  LAST_NEW_DAY = 18        // 最後一天背新字（Day 19–20 為模擬考收尾，不背新字）
  WORDS_PER_BATCH = 15
  INTERVALS = [1, 3, 7, 14]

每批單字
  「第 N 批」= 在 Day N 學的單字（N = 1..18）。

某天 D 的晨間回想批次
  reviewBatches(D) = INTERVALS 中每個 iv，取來源天 src = D - iv，
                     保留 1 <= src <= LAST_NEW_DAY 者。
  例：Day 10 → 回想 第9批(+1)、第7批(+3)、第3批(+7)
      Day 15 → 回想 第14批(+1)、第12批(+3)、第8批(+7)、第1批(+14)

每天的單字負擔
  新背 15 字（Day 1–18）＋ 回想 0–60 字（依到期批次數，第 15 天後達尖峰）
  全程新字總量 = 15 × 18 = 270 字
```

**回想方式**：英文 + 例句先顯示、中文意思隱藏，使用者先回想再點開驗證（輸出 > 輸入）。這個「努力回想」的動作本身就是記憶訊號，不可改成被動瀏覽。

## 4. 資料模型（目前 localStorage schema）

| key | 型別 | 內容 |
|---|---|---|
| `toeic20_start` | string | 開始日，格式 `YYYY-MM-DD` |
| `toeic20_tasks` | object | `{ "d{day}:{taskId}": true }`，只存已完成項 |
| `toeic20_vocab` | object | `{ "{day}": [ { w, m, exs:[{e,t},..], n, lo?, ri? }, ... ] }`。w=單字 m=中文 exs=例句陣列（每筆 `{e:英文, t:中文翻譯}`；舊的字串自動轉成 `{e, t:""}`）n=備註。`lo`=單字庫開始學習日(YYYY-MM-DD)、`ri`=已完成間隔階段(0..4)，僅單字庫頁面用、選填。鍵除了數字批次(第 N 批)外另有 `"lib"` 桶＝單字庫專屬、不屬任何 20 天批次。單字跨全表去重（trim+不分大小寫），重複輸入只把新例句補進既有的字、不另建、不重排批次。舊 `ex`(字串) 載入時自動轉成 `exs`。 |
| `toeic20_view` | string | 上次所在分頁：`sprint`(20 天衝刺) 或 `lib`(海馬迴單字庫)。 |
| 單字 `pr` 欄 | object | 逐字練習紀錄，存在每個單字物件內：`pr = { "YYYY-MM-DD": { c:答對, x:答錯 } }`。測驗答對／按「記得」記入 `c`，答錯／按「忘了」記入 `x`。隨 `vocab` 雲端同步。 |
| 單字 `lib` 欄 | string | 該字所屬的單字庫名稱（多單字庫用，預設「我的單字庫」）。只用於 `vocab["lib"]` 桶內的字。 |
| 單字 `da` 欄 | string | 加入日期 `YYYY-MM-DD`（不可變）。新增時設為當天；測驗依此分組，按「今天新學」不會改變它。舊資料回填為最早已知日期（`lo` 與 `pr` 中最早者）。 |
| `toeic20_libs` | array | 單字庫名稱清單（含尚未有字的空庫），隨雲端同步。 |
| `toeic20_curlib` | string | 目前選取的單字庫（裝置本機檢視狀態，不同步）。 |
| `toeic20_log` | object | （舊版）每日彙總對錯 `{ "YYYY-MM-DD": { c, x } }`。已改為逐字 `pr`，此鍵僅作為舊資料保留並併入每日總計圖表，不再寫入。 |

遷移到帳號版時，這三個結構直接對應後端的資料表 / 文件即可（見 §7）。

## 5. 學習計畫內容（領域知識，與程式邏輯分離）

### 五階段

| 區間 | 階段 | 重點 |
|---|---|---|
| Day 1–3 | 打地基 | 模擬考抓底、熟記 7 大 Part 配分、開始滾單字 |
| Day 4–9 | 聽力＋文法主攻 | CP 值最高，建立基礎聽力與文法直覺 |
| Day 10–14 | 擴張 Part 3/4/6 | 先看題再聽，抓關鍵資訊 |
| Day 15–18 | 攻 Part 7＋時間管理 | 關鍵字定位、控制每篇時間 |
| Day 19–20 | 全真模擬＋收尾 | 計時實戰、調節奏、複習錯題本 |

### 每日練習量

| 階段 | 聽力 | 文法 | 閱讀 | 單字 |
|---|---|---|---|---|
| Day 1 | 完整模擬考 1 回 | — | — | 開始背 15 |
| Day 2–3 | Part 1–2：20 題 + shadowing 5 句 | Part 5：10 題 + 訂正 | — | 15 新 + 複習 |
| Day 4–9 | Part 1–2：25–30 題 + shadowing 8 句 | Part 5：15 題 + 歸類訂正 | — | 15 新 + 複習 |
| Day 10–14 | Part 3/4：3–4 篇（約 10–12 題） | Part 5/6：10–15 題 | — | 15 新 + 複習 |
| Day 15–18 | Part 3/4：2 篇維持手感 | Part 5/6：10 題 | Part 7：單篇 2–3 篇（計時） | 15 新 + 複習 |
| Day 19–20 | 全真計時模擬 1 回 | （含模擬考內） | （含模擬考內） | 不背新字，複習錯題本 |

**判斷量夠不夠的標準**：不是題數，是有沒有「訂正歸類」到位（詞性 / 時態 / 介系詞 / 連接詞）。
**1 小時縮減版**：保留 單字＋聽力＋文法 三項核心、量打八折，砍掉閱讀；間隔複習絕不能斷。

### 海馬迴四原則

1. **輸出 > 輸入**：蓋住中文、考自己。
2. **間隔複習**：+1 / +3 / +7 / +14 天重新喚起。
3. **睡眠固化**：新字排睡前、起床先回想，別熬夜。
4. **連結 + 情緒**：新字連到已知事物（工作上的 certification / compliance / specification 等幾乎免背）。

## 7. 帳號與同步：架構選項（M3 決策用）

前提：GitHub Pages 是純靜態代管，**跑不了登入 / 資料庫**。要帳號 = 要後端。三條路：

### 路線 0｜不做帳號，只做匯出/匯入（= M2）
零後端、零成本、純靜態。換裝置需手動匯出/匯入一個小檔。對單人衝刺工具常常已足夠。

### 路線 1｜Supabase（**推薦，CP 值最高**）
- 託管 PostgreSQL + 內建 Auth（Email/密碼），有免費方案。
- 前端續放 GitHub Pages，用 JS SDK 登入；以 Row Level Security 讓每帳號只讀寫自己的列。
- 不用自管伺服器、不用自寫驗證；底層是 Postgres（自己熟），約一個下午可接起。
- 建議資料表：
  - `profiles(id, ...)`
  - `settings(user_id, start_date)`
  - `tasks(user_id, day, task_id, done)`
  - `vocab(user_id, day, idx, word, meaning, example)`

### 路線 2｜自架 Django REST Framework 後端（最貼技術棧，但最重）
- DRF + PostgreSQL + Docker + NGINX，JWT/Token 驗證；前端 GitHub Pages 跨網域呼叫（需處理 CORS）。
- 掌控度最高、資料完全自有，且這套帳號/驗證邏輯與「創作者平台」同類，可複用。
- 代價：要一台常開的機器 + HTTPS。對讀書工具偏重。

**建議**：M2（匯出/匯入）先上、今天就能用；真的要「自動同步 + 正式帳號」再上路線 1。路線 2 留給想練手或想與創作者平台共用驗證時。

## 8. 專案結構（現況）

為了加入第二個頁面（單字庫），已把原本內嵌單檔拆成共用檔案（純靜態、無建置）：

```
Toeic_practise/
├── index.html          # 20 天衝刺頁（GitHub Pages 首頁）
├── library.html        # 海馬迴單字庫頁（獨立網址，與衝刺頁共用資料）
├── dashboard.html      # 練習紀錄頁（所有單字庫合計）
├── styles.css          # 共用樣式（兩頁 <link>）
├── app-core.js         # 共用主邏輯：狀態/SRS/單字helpers/各區渲染（缺的容器自動略過）
├── sync.js             # 共用 Firebase RTDB 同步層（ES module，透過 window.TOEIC 橋接）
├── Docs/PROJECT_PLAN.md / FIREBASE_SETUP.md
└── .claude/launch.json # 本機預覽用（未進版控）
```

關鍵設計：
- **共用一份 `app-core.js`**：包含全部邏輯，每個渲染函式只在對應容器存在時才動作，所以同一支腳本能同時服務兩個頁面、無重複。
- **`window.TOEIC` 橋接**：`app-core.js` 暴露 `getLocal / applyRemote / isLocalEmpty / setOnChange`，`sync.js` 只依賴這個介面 → 換後端（如 Supabase）時改 `sync.js` 即可，UI/邏輯不動（即原規劃的「儲存抽象層」精神）。
- 兩頁透過 `localStorage`（＋登入後 RTDB）共用同一份 `vocab`，達成「共用單字池、各自排程」。

> 未來若要做單元測試，可再把間隔複習純函式（`recallBatches` / `libStatus` 等）獨立成 `srs.js`。
